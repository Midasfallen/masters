## 1. User Flow

### 1.1 Общий флоу приложения

```
Запуск приложения
    │
    ├─→ Первый запуск?
    │   ├─ ДА → Onboarding (3 экрана)
    │   │         │
    │   │         └─→ Выбор роли (Клиент / Мастер)
    │   │               │
    │   │               └─→ Регистрация/Вход
    │   │                     │
    │   │                     └─→ Главный экран
    │   │
    │   └─ НЕТ → Проверка авторизации
    │               ├─ Авторизован → Главный экран
    │               └─ Не авторизован → Экран входа
    │
    └─→ Главный экран
          │
          ├─→ Bottom Navigation (5 вкладок)
          │   ├─ Home (Поиск и рекомендации)
          │   ├─ Search (Поиск с фильтрами)
          │   ├─ Favorites (Избранное)
          │   ├─ Bookings (Мои записи)
          │   └─ Profile (Профиль)
          │
          └─→ Действия пользователя
              ├─ Поиск мастера
              ├─ Просмотр профиля
              ├─ Создание записи
              ├─ Чат с мастером
              └─ Оставить отзыв
```

### 1.2 Флоу клиента (запись к мастеру)

```
1. Главный экран
   │
2. Поиск мастера
   ├─ По категории (💇 Парикмахер)
   ├─ По геолокации (Рядом)
   └─ По имени (Search)
   │
3. Список мастеров
   ├─ Фильтры (Цена, Рейтинг, Расстояние)
   └─ Сортировка (По рейтингу, По расстоянию, По цене)
   │
4. Профиль мастера
   ├─ Портфолио (фото/видео)
   ├─ Услуги и цены
   ├─ Отзывы (234)
   └─ О мастере
   │
5. Выбор услуги
   ├─ Одна услуга
   └─ Несколько услуг (комбо)
   │
6. Выбор даты и времени
   ├─ Календарь
   └─ Доступные слоты
   │
7. Подтверждение записи
   ├─ Детали: Услуга, Дата, Время, Цена
   └─ Комментарий (опционально)
   │
8. Запись создана ✅
   ├─ Уведомление мастеру
   ├─ Push уведомление клиенту
   └─ Напоминание за 1 час до записи
   │
9. После визита
   └─ Оставить отзыв (Push через 2 часа)
```

### 1.3 Флоу мастера (управление профилем)

```
1. Регистрация как мастер
   │
2. Заполнение профиля
   ├─ Фото профиля
   ├─ Категория и специализация
   ├─ О себе (био)
   ├─ Контакты (скрыты до записи)
   └─ Локация (адрес или радиус работы)
   │
3. Добавление услуг
   ├─ Название
   ├─ Описание
   ├─ Цена
   ├─ Длительность
   └─ Фото (привязка к портфолио)
   │
4. Настройка расписания
   ├─ Рабочие дни и часы
   ├─ Перерывы
   ├─ Выходные / Отпуск
   └─ Длительность слотов (30 мин, 1 час, custom)
   │
5. Портфолио
   ├─ Загрузка фото (до 50 шт)
   ├─ Загрузка видео (до 15 сек, 10 шт)
   ├─ Описание работ
   └─ Привязка к услугам
   │
6. Управление записями
   ├─ Новые (требуют подтверждения)
   ├─ Подтверждённые
   ├─ Завершённые
   └─ Отменённые
   │
7. Премиум подписка (опционально)
   └─ Приоритет в поиске, Аналитика, Без рекламы
```

## 3. Навигация

### 3.1 Типы навигации

#### **Bottom Navigation** (Основная)
- 5 вкладок: Home, Search, Favorites, Bookings, Profile
- Всегда видна на главных экранах
- Animated icon transitions
- Badge для уведомлений (на Bookings, Profile)

#### **Stack Navigation** (Вложенная)
- Для detail экранов
- AppBar с кнопкой "Назад"
- Hero transitions для изображений

#### **Modal Navigation** (Модальные окна)
- Bottom Sheets для фильтров, действий
- Dialogs для подтверждений
- Full-screen modals для сложных форм

#### **Drawer Navigation** (Боковое меню)
- Не используется в MVP
- Заменено на Bottom Navigation

### 3.2 Transitions (Переходы)

```dart
// Default transition
pageTransitionsTheme: PageTransitionsTheme(
  builders: {
    TargetPlatform.android: ZoomPageTransitionsBuilder(),
    TargetPlatform.iOS: CupertinoPageTransitionsBuilder(),
  },
),

// Hero transition для изображений
Hero(
  tag: 'master-avatar-${masterId}',
  child: CircleAvatar(...),
)

// Shared axis transition для табов
SharedAxisTransition(
  animation: animation,
  secondaryAnimation: secondaryAnimation,
  transitionType: SharedAxisTransitionType.horizontal,
  child: child,
)
```

## 4. Детальное описание экранов

### 4.1 SPLASH SCREEN (Заставка)

**Цель:** Показать бренд пока грузится приложение

**Layout:**
```
┌────────────────────────┐
│                        │
│                        │
│        [LOGO]          │ ← Центрировано
│     SERVICE            │
│     Platform           │
│                        │
│   [Loading spinner]    │ ← Primary color
│                        │
└────────────────────────┘
```

**Элементы:**
- Логотип (svg, масштабируемый)
- Название приложения (Typography: Display Medium)
- Loading indicator (CircularProgressIndicator)
- Фон: Gradient (Primary → Secondary)

**Анимация:**
- Fade in логотипа (300ms)
- Pulse loading indicator
- Автоматический переход (max 2 секунды)

**Transitions:**
- → Onboarding (первый запуск)
- → Login (не авторизован)
- → Home (авторизован)

## 5. UX Patterns

### 5.1 Pull-to-Refresh

**Где используется:**
- Home feed
- Search results
- Bookings list
- Chat list

**Анимация:**
- Circular progress indicator
- Показывать после 50px pull
- Vibration feedback при refresh

### 5.2 Infinite Scroll

**Где используется:**
- Feed
- Search results (список мастеров)
- Portfolio gallery

**Поведение:**
- Загружать следующую страницу за 3 items до конца
- Loading indicator внизу списка
- "Больше нечего показать" если достигнут конец

### 5.3 Empty States

**Принципы:**
- Иконка (большая, 80dp)
- Заголовок (что пусто)
- Описание (почему пусто)
- CTA кнопка (что делать)

**Примеры:**
```
No Bookings:
  📅 Icon
  "У тебя пока нет записей"
  "Найди мастера и запишись на удобное время"
  [Найти мастера]

No Favorites:
  ❤️ Icon
  "Ничего не сохранено"
  "Сохраняй понравившихся мастеров"
  [На главную]

No Search Results:
  🔍 Icon
  "Ничего не найдено"
  "Попробуй изменить фильтры или поисковый запрос"
  [Сбросить фильтры]
```

### 5.4 Error States

**Типы ошибок:**

**Network Error:**
```
📡 Icon
"Нет интернета"
"Проверь подключение и попробуй снова"
[Повторить]
```

**Server Error:**
```
⚠️ Icon
"Что-то пошло не так"
"Мы уже работаем над этим. Попробуй позже"
[На главную]
```

**Validation Error:**
```
Inline под полем:
"Email должен быть в формате example@mail.com"
Цвет: Error color
```

### 5.5 Loading States

**Skeleton Screens:**
- Для списков (карточки мастеров)
- Shimmer эффект
- Серый градиент

**Progress Indicators:**
- Circular для общей загрузки
- Linear для прогресса (upload фото)

### 5.6 Swipe Actions

**Bookings:**
- Swipe left → "Отменить" (красная кнопка)
- Swipe right → "Написать" (синяя кнопка)

**Chats:**
- Swipe left → "Удалить" (красная)
- Swipe right → "Архив" (серая)

### 5.7 Bottom Sheets

**Используются для:**
- Фильтры (Search)
- Выбор из списка (сортировка)
- Быстрые действия (share, report)
- Выбор даты/времени (альтернатива модалу)

**Характеристики:**
- Draggable handle сверху
- Закрытие свайпом вниз
- Backdrop (затемнение 40%)

## 6. Responsive Design

### 6.1 Breakpoints

```
Mobile:    < 600dp     (1 column)
Tablet:    600-840dp   (2 columns)
Desktop:   > 840dp     (3 columns + sidebar)
```

### 6.2 Adaptive Layouts

**Phone (Portrait):**
- Single column
- Bottom navigation
- Full-width cards

**Tablet:**
- Two columns для списков
- Master-Detail pattern
- Navigation rail вместо bottom nav

**Desktop/Web:**
- Three columns
- Persistent navigation drawer
- Wider content max-width: 1200px

---

## 7. Accessibility

### 7.1 Screen Reader

- Все кнопки: описательные labels
- Изображения: alt text / semantic labels
- Формы: связанные labels
- Announce для dynamic контента

### 7.2 Keyboard Navigation

- Tab между полями
- Enter для submit
- Escape для закрытия модалов
- Arrow keys для lists (опционально)

### 7.3 Touch Targets

- Минимум 48dp × 48dp
- Spacing между targets: 8dp
- Кнопки: легко попадать пальцем
