# Настройка Flutter Web для разработки

## Версия Flutter

**Текущая версия:** Flutter 3.35.3

## Запуск на Web

### Базовая команда
```bash
cd frontend
flutter run -d chrome
```

### Важные изменения в Flutter 3.35+

В версии Flutter 3.35+ флаг `--web-renderer` был **удален**. Рендерер теперь выбирается автоматически:
- **По умолчанию** используется **CanvasKit** для лучшей поддержки шрифтов и производительности
- HTML рендерер используется только при необходимости (например, для очень старых браузеров)

### Проверка доступных устройств
```bash
flutter devices
```

Должен показать Chrome в списке доступных устройств:
```
Chrome (chrome) • chrome • web-javascript • Google Chrome ...
```

### Альтернативные команды

Если Chrome не найден автоматически:
```bash
# Указать конкретный браузер
flutter run -d web-server --web-port 8080
```

Затем открыть вручную: `http://localhost:8080`

### Отладка

Для отладки с DevTools:
```bash
flutter run -d chrome --debug
```

DevTools откроется автоматически или будет доступен по ссылке в консоли.

## Известные проблемы

### Проблема: "Could not find an option named '--web-renderer'"

**Причина:** Использование устаревшего флага из старых версий Flutter.

**Решение:** Убрать флаг `--web-renderer` из команды. Использовать просто:
```bash
flutter run -d chrome
```

### Проблема: Chrome не найден

**Решение:**
1. Убедиться, что Chrome установлен
2. Проверить PATH: Chrome должен быть доступен из командной строки
3. Использовать `flutter run -d web-server` как альтернативу

---

## Рекомендации для разработки

1. **Всегда использовать Chrome** для разработки (лучшая поддержка DevTools)
2. **Проверять на других браузерах** перед коммитом (Firefox, Safari, Edge)
3. **Использовать DevTools** для отладки Network запросов и производительности
4. **Тестировать с throttling** (Fast 3G) для проверки оптимистичного UI
