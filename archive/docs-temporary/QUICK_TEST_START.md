# Быстрый старт для тестирования

## Команды для запуска

### 1. Backend
```bash
cd backend
npm run start:dev
```

### 2. Frontend (Web)
```bash
cd frontend
flutter pub get
flutter run -d chrome
```

**Примечание:** В Flutter 3.35+ флаг `--web-renderer` удален. Рендерер выбирается автоматически (по умолчанию используется CanvasKit для лучшей поддержки шрифтов).

### 3. Открыть DevTools
- Нажать `F12` в Chrome
- Перейти на вкладку **Network**
- Установить **Throttling → Fast 3G** (для тестирования оптимистичного UI)

## Быстрая проверка основных функций

### ✅ Загрузка фото (30 секунд)
1. Открыть экран создания поста
2. Выбрать любое фото
3. Проверить, что фото отображается в предпросмотре
4. Создать пост
5. **DevTools → Network:** Проверить, что в запросе есть `filename="something.jpg"`

### ✅ Лайки (20 секунд)
1. Открыть любой пост
2. Быстро кликнуть на лайк 2-3 раза
3. Проверить, что счетчик изменился только один раз
4. **DevTools → Network:** Установить "Offline"
5. Нажать лайк → должен произойти откат с ошибкой

### ✅ Поделиться (10 секунд)
1. Открыть любой пост
2. Нажать "Поделиться"
3. Должен появиться SnackBar "Ссылка скопирована в буфер обмена"
4. Вставить ссылку в адресную строку (Ctrl+V)

### ✅ Аватары (15 секунд)
1. Найти пользователя без фото
2. Запомнить цвет аватара
3. Перезагрузить страницу (F5)
4. Цвет должен остаться тем же

## Критические проверки перед коммитом

- [ ] `flutter analyze` - нет ошибок
- [ ] Загрузка фото работает на Web
- [ ] Лайки работают с оптимистичным UI
- [ ] Share работает с fallback
- [ ] Аватары отображаются корректно

**Время полного тестирования:** ~5-10 минут
